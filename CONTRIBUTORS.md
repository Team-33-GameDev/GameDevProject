## Contribution Principles

Contributions should be traceable through one or more of:

- Git commits;
- pull requests and reviews;
- GitHub issues;
- architecture or project documentation;
- art/audio asset history;
- sprint reports and review evidence.

A person should not be credited solely because their name appears in an automatically generated document. The team should verify this file before final submission.

## Current Collaboration Workflow

1. Work is tracked in a GitHub issue.
2. A contributor creates a focused feature branch.
3. The change is implemented and tested.
4. A pull request is opened.
5. A peer reviews the change.
6. The change is merged into `main` after checks and acceptance criteria are satisfied.

See [`docs/development-process.md`](docs/development-process.md) and [`docs/definition-of-done.md`](docs/definition-of-done.md).

## Third-Party Assets and External Credits

External assets, fonts, sounds, textures, models, and other third-party materials are not team contributors. Record their authors, sources, and licenses in:

- [`ATTRIBUTION.md`](ATTRIBUTION.md)

When adding a third-party asset:

1. confirm that the license permits the intended use;
2. record the source and author;
3. record the license;
4. retain any required notice;
5. avoid assets with unclear redistribution rights.

## AI-Assisted Work

AI systems are tools, not human contributors. AI-assisted drafting or coding must be disclosed where required in:

- [`reports/week6/llm-report.md`](reports/week6/llm-report.md)

Human contributors remain responsible for reviewing and validating generated content.
