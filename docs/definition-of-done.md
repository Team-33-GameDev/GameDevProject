# Definition of Done

The following criteria must be met for any Product Backlog Item (PBI) to be considered "Done" and ready for release:

## Code Quality & Integration
- Code is written, reviewed via Pull Request, and merged into the `main` branch.
- No critical or high-severity bugs are introduced.
- All existing automated tests pass, and new tests are written for new functionality (where applicable).

## Testing & Verification
- The feature has been manually tested on the target platform (PC/Windows).
- The feature works exactly as described in the Acceptance Criteria.

## Documentation & Release
- `CHANGELOG.md` is updated with a clear, user-visible description of the change for every user-facing feature, improvement, or bug fix.
- Any necessary updates to the `README.md` or player documentation are made.
