# LLM Usage Report (Week 7)

This document discloses the use of AI and Large Language Model (LLM) tools during Week 7 of the project. 

## Tools Used
- **GitHub Copilot**: For minor code autocompletion and boilerplate generation.
- **ChatGPT**: For Markdown formatting, grammar checking, and generating initial document templates.

## How AI/LLM Tools Were Used
1. **Documentation Templates**: LLMs were used to generate the initial Markdown structure and boilerplate headings for weekly reports (e.g., `reports/week7/README.md`, `retrospective.md`, and `reflection.md`) to ensure we did not miss any required sections from the Assignment 6 guidelines.
2. **Proofreading and Clarity**: Final drafts of the Sprint Review summary, customer handover documentation, and reflection artifacts were passed through an LLM to check for grammatical errors, improve sentence clarity, and ensure a professional tone.
3. **Minor Code Autocomplete**: GitHub Copilot was used occasionally for standard, repetitive GDScript boilerplate (e.g., basic signal connections or UI node references), which was then manually reviewed and tested by the implementing developer.

## Original Team Effort
All core product logic, architecture decisions (ADRs), customer feedback analysis, gameplay balancing, and the final content of all reports were authored, critically evaluated, and validated entirely by the human team members. AI tools were used strictly as mechanical assistants for formatting and boilerplate reduction, and no substantive project decisions or original analysis were delegated to an LLM.
